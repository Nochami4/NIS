# НИС: Моделирование случайных процессов с помощью моделей авторегрессии

## Тема

Моделирование случайных процессов с помощью моделей авторегрессии.

## Идея проекта

Проект посвящён исследованию свойств случайного процесса (временного ряда средней суточной температуры воздуха в Волгограде) по историческим данным Meteostat. Объект исследования — суточный температурный ряд, в котором требуется выделить временную зависимость, сезонную компоненту и лаговую структуру.

Цель работы — анализ структуры временного ряда и получение прогноза его будущих значений методами авторегрессии (с учётом лагов и сезонности).

Работа построена как научно-исследовательский пайплайн:

1. Из почасовых и суточных данных Meteostat формируется воспроизводимый набор данных с суточным разрешением;
2. Создаются календарные, лаговые и скользящие признаки для описания структуры случайного процесса;
3. Проводится моделирование и сравнение простых статистических ориентиров (`Naive`, `Seasonal naive`) с полноценными авторегрессионными моделями разной глубины;
4. Отдельно исследуется, как меняется предсказательная способность моделей при изменении длины обучающей выборки (при фиксированных валидационной и тестовой выборках).

Итог работы — сравнение авторегрессионных спецификаций и базовых ориентиров при прогнозировании общего временного ряда.

## Что сравнивается

В модуле моделирования рассматриваются:

- `Naive (t-1)` — простой авторегрессионный ориентир по значению предыдущего дня;
- `Seasonal naive (t-365)` — сезонный ориентир по значению примерно того же дня прошлого года;
- `AR(1)`, `AR(7)` и `AR(30)` — авторегрессионные модели с одним лагом;
- `AR с короткими лагами` — авторегрессионная модель с лагами 1, 2, 3, 7, 14 и 30;
- `AR с сезонным лагом` — авторегрессионная модель с короткими лагами и лагом 365.

Простые модели задают понятные статистические точки отсчёта, с которыми сравниваются авторегрессионные модели, учитывающие историю и сезонность процесса.

Основные метрики:

- `MAE`
- `RMSE`
- `MASE`
- `bias`

## Воспроизводимость

В проекте зафиксированы:

- конечная дата используемого набора данных: `2025-12-31`;
- валидационная выборка: `2024-01-01 ... 2024-12-31`;
- тестовая выборка: `2025-01-01 ... 2025-12-31`.

Все эти настройки собраны в `src/constants.py`.
