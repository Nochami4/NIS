# Курсовой проект

## Тема

Применение нейросетей для статистической обработки информации.

## Идея проекта

Проект посвящён статистической обработке временного ряда средней суточной температуры воздуха в Волгограде по историческим данным Meteostat. Объект исследования — дневной температурный ряд, в котором нужно выделить временную зависимость, сезонность и лаговую структуру.

Цель работы — получить прогноз будущих значений ряда по его прошлым значениям и календарным признакам, а затем сравнить ошибки классических и нейросетевых подходов на одной и той же задаче. Нейросети здесь не заменяют статистический анализ, а применяются как гибкие модели для обработки временной зависимости и сезонной информации.

Работа построена как учебно-исследовательский пайплайн:

1. из почасовых и суточных данных Meteostat формируется воспроизводимый дневной датасет;
2. создаются календарные, лаговые и скользящие признаки для описания структуры временного ряда;
3. затем сравниваются конкретные модели: `Naive`, `Seasonal naive`, авторегрессионные модели, `RandomForest`, `Dense`, `GRU`, `Conv1D`;
4. отдельно исследуется, как меняется качество моделей при разной длине обучающей выборки при фиксированных validation и test интервалах.

Итог работы — не утверждение о безусловном превосходстве нейросетей, а сравнение статистических ориентиров, вспомогательной табличной модели и нейросетевых моделей на общем временном ряде.

## Что сравнивается

В ноутбуке моделирования рассматриваются:

- `Naive (t-1)` — простой авторегрессионный ориентир по значению предыдущего дня;
- `Seasonal naive (t-365)` — сезонный ориентир по значению примерно того же дня прошлого года;
- `AR(1)`, `AR(7)` и `AR(30)` — авторегрессионные модели с одним лагом;
- `AR с короткими лагами` — авторегрессионная модель с лагами 1, 2, 3, 7, 14 и 30;
- `AR с сезонным лагом` — авторегрессионная модель с короткими лагами и лагом 365;
- `RandomForestRegressor` — вспомогательная табличная модель для сравнения с нейросетями на тех же лаговых и календарных признаках;
- `Dense`-сеть — нейросетевая модель по лаговым, скользящим и календарным признакам;
- `GRU` — нейросетевая модель по последовательностным окнам длиной 30 дней;
- `Conv1D` — нейросетевая модель по тем же временным окнам.

Простые и авторегрессионные модели задают понятные статистические точки отсчёта. Нейросети `Dense`, `GRU` и `Conv1D` рассматриваются как более гибкие модели, которые могут учиться использовать лаговую и сезонную информацию из временного ряда.

Основные метрики:

- `MAE`
- `RMSE`
- `MASE`
- `bias`

## Воспроизводимость

В проекте зафиксированы:

- конечная дата используемого датасета: `2025-12-31`;
- validation-интервал: `2024-01-01 ... 2024-12-31`;
- test-интервал: `2025-01-01 ... 2025-12-31`;
- random seed для моделей и нейросетей.

Все эти настройки собраны в `src/constants.py`.

## Подготовка окружения

### 1. Создание виртуального окружения

Linux/macOS/WSL:

```bash
python3 -m venv .venv
```

Если команда `python3 -m venv .venv` не работает в Ubuntu/WSL, нужно установить пакет `python3-venv` или версию `python3.X-venv`, соответствующую вашей версии Python.

Windows:

```bash
python -m venv .venv
```

### 2. Активация виртуального окружения

Windows PowerShell:

```powershell
.venv\Scripts\Activate.ps1
```

Windows cmd:

```cmd
.venv\Scripts\activate.bat
```

Linux/macOS/WSL:

```bash
source .venv/bin/activate
```

```bash
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

## Порядок запуска

1. Запустите `notebooks/01_dataset_creation.ipynb`.
2. Убедитесь, что создан файл `data/processed/volgograd_daily_temperature_dataset.csv`.
3. Затем запустите `notebooks/02_modeling_and_comparison.ipynb`.

Важно: второй ноутбук опирается на результаты первого.

## Что делает первый ноутбук

`notebooks/01_dataset_creation.ipynb`:

- загружает данные Meteostat для Волгограда;
- пытается выбрать ближайшую подходящую станцию;
- при сетевой ошибке может использовать сохранённые raw-файлы из `data/raw`;
- агрегирует hourly-данные до уровня дней;
- объединяет hourly-признаки и fallback из `Daily`;
- создаёт календарные, лаговые и скользящие признаки;
- аккуратно обрабатывает пропуски без использования будущих данных;
- сохраняет итоговый датасет в `data/processed/volgograd_daily_temperature_dataset.csv`.

## Что делает второй ноутбук

`notebooks/02_modeling_and_comparison.ipynb`:

- загружает подготовленный датасет и ограничивает его фиксированным периодом;
- в начале автоматически удаляет устаревшие `02_*` артефакты предыдущих версий из `outputs/figures` и `outputs/tables`;
- проверяет покрытие по годам и формирует сценарии длины train;
- выполняет предварительный анализ временного ряда на обучающем периоде;
- оценивает авторегрессионные модели в режиме последовательного однодневного прогноза;
- сравнивает модели при одинаковых validation и test интервалах;
- считает `MAE`, `RMSE`, `MASE` и `bias`;
- строит итоговые таблицы на validation и test;
- делает отдельный анализ качества при разной длине train;
- строит графики прогнозов, кривых обучения и сезонных ошибок;
- формулирует осторожные исследовательские выводы по реально посчитанным таблицам.

## Выходные таблицы и графики

Основные таблицы в `outputs/tables/`:

- `02_yearly_coverage.csv` и `02_train_scenarios.csv` — покрытие данных по годам и сценарии длины train;
- `02_adf_stationarity.csv` — результат проверки стационарности через тест Дики — Фуллера на обучающем периоде;
- `02_ar_lag_selection.csv` — сравнение авторегрессионных спецификаций по `AIC` и `BIC`;
- `02_ar_forecast_checks.csv` — проверка длины и полноты авторегрессионных прогнозов;
- `02_train_size_validation_metrics.csv` и `02_train_size_test_metrics.csv` — метрики при разной длине обучающей истории;
- `02_train_size_neural_advantage.csv` — сравнение лучшей нейросети и `RandomForest` по сценариям;
- `02_main_validation_metrics.csv` и `02_main_test_metrics.csv` — итоговое сравнение моделей на validation и test;
- `02_main_test_predictions.csv` — прогнозы моделей на test-интервале;
- `02_random_forest_timeseriessplit.csv` — дополнительная проверка устойчивости `RandomForest`;
- `02_seasonal_error_analysis.csv` — ошибки моделей по сезонам.

Основные графики в `outputs/figures/`:

- `01_temperature_time_series.png` и `01_monthly_seasonality.png` — первичный анализ подготовленного ряда;
- `02_acf_pacf_temperature.png` — автокорреляция и частичная автокорреляция температуры на обучающем периоде;
- `02_rolling_mean_std_temperature.png` — скользящее среднее и скользящее стандартное отклонение температуры;
- `02_train_size_mae.png` — влияние длины train на `MAE`;
- `02_forecast_overview_key_models.png` — обзор факта и ключевых моделей на тестовом периоде;
- `02_forecast_small_multiples.png` — отдельные графики прогноза для каждой модели;
- `02_residuals_key_models.png` — ошибки прогноза для ключевых моделей;
- `02_mae_by_model.png` — сравнение моделей по `MAE`;
- `02_mae_by_season_heatmap.png` — сезонное сравнение ошибок по `MAE`;
- `02_neural_learning_curves.png` — кривые обучения нейросетей.
